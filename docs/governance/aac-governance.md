# AAC Governance Model (§8)

## Working Group Composition

| Stakeholder Class | Seats | Role |
|---|---|---|
| AI providers (Tier 1) | 2 (rotating) | Spec implementers; primary funders |
| AI providers (Tier 2) | 1 | Diverse perspective |
| Enterprise publishers | 2 | High-volume content creators |
| Independent creators | 1 | Non-enterprise interests |
| Developer community | 1 | Tooling and SDK feedback |
| Regulators (non-voting) | — | Regulatory alignment |
| Civil society | 1 | End-user and privacy interests |
| Independent chair | 1 | Tie-breaking, agenda-setting |

## Specification Change Process

1. Working group member submits proposal with rationale and implementation assessment
2. 60-day public comment period (open to all)
3. Working group discussion and revision
4. Consensus vote (6/8 supermajority for MAJOR changes; simple majority for MINOR)
5. 90-day implementation notice before new version takes effect
